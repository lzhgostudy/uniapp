# 应用声明周期

|函数名|说明|
|:-|:-|
|onLaunch|应用初始化完成时触发（全局执行一次）|
|onShow|应用启动，或从后台进入前台显式|
|onHide|应用从前台进入后台|
|onError|应用报错时触发|
|onUniNViewMessage|对nvue页面发送的数据进行监听|
|onUnhandleRejection|对未处理的Promise拒绝事件监听函数|
|onPageNotFound|页面不存在监听函数|
|onThemeChange|监听系统主题变化|

**注意**
- 应用生命周期仅可在`App.vue`中监听，在其他页面监听无效。

```js
export default {
	onLaunch: function() {
		
	},
	onShow: function() {
		
	},
	onHide: function() {
		
	}
}
```


# 页面生命周期

|函数名|说明|
|:-|:-|
|onInit|监听页面初始化，其参数同onLoad参数，为上个页面传递的数据，参数类型为Object(用于页面传参)，触发时机早于onLoad|
|onLoad|监听页面加载，其参数为上个页面传递的数据，参数类型为Object|
|onShow|监听页面显示，页面每次出现在屏幕上都会触发，包括从下级页面返回当前页面时|
|onReady|监听页面初次渲染完成。注意如果渲染速度快，会在页面进入动画完成前触发|
|onHide|监听页面隐藏|
|onUnload|监听页面卸载|
|onResize|监听窗口尺寸变化|
|onPullDownRefresh|监听用户下拉动作，一般用于下拉刷新|
|onReachBottom|页面滚动到底部的事件（不是scroll-view滚到底），常用下拉下一页数据。具体见下方注意事项|
|onTabItemTap|点击tab时触发，参数为Object|
|onShareAppMessage|用户点击分享|
|onPageScroll|监听页面滚动，参数Object|
|onNavigationBarButtonTap|监听原生标题栏按钮点击事件，参数为Object|
|onBackPress|监听页面返回，返回event = { from:backbutton, navigateBack }, backbutton 表示来源是左上角返回按钮或android返回键；navigateBack表示来源是uni.navigateBack|

**`onInit`使用注意**
- 不依赖页面传参的逻辑可以直接使用created生命周期代替
